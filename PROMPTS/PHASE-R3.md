# Phase R3 — JavaScript Purification

## Goal

Reduce hand-written JavaScript in the app by moving maintainable application
logic into ClojureScript where it naturally belongs, while leaving generated
files and browser boundary modules in JavaScript.

This is a refactor phase. It should not change user-facing behaviour, plugin
semantics, audio output, syntax highlighting behaviour, completions, hover docs,
or bundle layout except where explicitly called out below.

**Motivation:** REPuLse is a ClojureScript app with a Lisp and pattern engine in
CLJS, but several islands of hand-written JavaScript remain. Some are legitimate
browser/API boundaries; others are application logic that would be easier to
maintain, test, and evolve in CLJS. This phase purifies the codebase without
forcing CLJS into places where it makes the system worse.

---

## Classification

### Convert to CLJS

The best first target is the CodeMirror language support island:

- `app/src/repulse/lisp-lang/completions.js`
- `app/src/repulse/lisp-lang/defs-completion.js`
- `app/src/repulse/lisp-lang/highlight.js`
- `app/src/repulse/lisp-lang/hover.js`
- `app/src/repulse/lisp-lang/index.js`
- `app/src/repulse/lisp-lang/insert-categories.js`
- `app/src/repulse/lisp-lang/insert-helper.js`
- `app/src/repulse/lisp-lang/keywords-completion.js`
- `app/src/repulse/lisp-lang/providers.js`
- `app/src/repulse/lisp-lang/rainbow.js`

These files are editor/application logic and are already consumed by CLJS.
Porting them removes the largest hand-written JS island without changing the
audio engine or external plugin story.

### Keep as JS

Do not convert generated or boundary files:

- `app/src/repulse/lisp-lang/parser.js` — generated from Lezer grammar
- `app/src/repulse/lisp-lang/parser.terms.js` — generated from Lezer grammar
- `app/public/repulse_audio.js` and `packages/audio/pkg/repulse_audio.js` — generated by `wasm-pack`
- `app/public/embed.js` — generated by Shadow CLJS from `repulse.embed`
- `app/externs/*.js` — Closure extern declarations
- `app/public/worklet.js` — AudioWorklet bootstrap boundary
- `app/public/worklets/*.js` — AudioWorkletProcessor modules
- `app/public/plugin-base.js` — public JS authoring API for third-party plugins
- `app/public/plugins/*.js` — built-in examples and external-plugin compatibility surface

Plugin files may be revisited later, but they should not be converted in this
phase. Keeping them as readable JS examples is useful for external authors.

## Files to change

| File | Change |
|------|--------|
| `app/src/repulse/lisp_lang/*.cljs` or `app/src/repulse/lisp-lang/*.cljs` | New CLJS equivalents for CodeMirror language support |
| `app/src/repulse/ui/editor.cljs` | Import CLJS language extension instead of JS `index.js` |
| `app/src/repulse/app.cljs` | Import CLJS provider setters instead of JS `providers.js` |
| `shadow-cljs.edn` | Adjust source/module config only if needed |
| `app/src/repulse/lisp-lang/*.js` | Remove converted hand-written JS files after parity is verified |
| `docs/ARCHITECTURE.md` | Document which JS files are intentional boundaries |

Namespace naming is part of the implementation decision. If the existing
hyphenated directory causes awkward CLJS namespace names, introduce
`app/src/repulse/lisp_lang/` and leave only generated Lezer output under
`lisp-lang/`.

## Migration plan

1. Move provider state first (`setBankNamesProvider`, `setFxNamesProvider`) and
   update `app.cljs`.
2. Port data-only modules (`completions`, `insert-categories`, hover docs).
3. Port CodeMirror extension modules (`highlight`, `rainbow`, `insert-helper`,
   completion sources).
4. Replace `lispLanguage` import in `ui/editor.cljs`.
5. Delete only the hand-written JS files that are fully replaced.
6. Run grammar generation only if the grammar changed. This phase should not
   require grammar changes.

Prefer small parity-preserving commits if this phase is implemented manually.
The old and new language extensions should not coexist permanently.

---

## Definition of done

- [ ] All hand-written CodeMirror language-support JS listed above has CLJS equivalents
- [ ] `repulse.ui.editor` consumes the CLJS `lisp-language` extension directly
- [ ] `repulse.app` updates bank/effect provider functions through CLJS
- [ ] Syntax highlighting, bracket matching, rainbow parens, completions, live `def` completion, hover docs, diagnostics, and contextual insertion still work
- [ ] Generated `parser.js` / `parser.terms.js` remain generated JS and are not manually edited
- [ ] `app/public/plugin-base.js`, `app/public/plugins/*.js`, and worklets remain JS
- [ ] `npm run test` passes
- [ ] `npx shadow-cljs compile app` passes without new warnings
- [ ] Manual browser smoke test confirms editor load, typing, hover, completion, and `+` insertion behaviour
- [ ] `docs/ARCHITECTURE.md` explains the remaining intentional JS boundaries

---

## Out of scope

- No conversion of generated files
- No conversion of AudioWorklet modules
- No conversion of the third-party plugin API
- No changes to REPuLse-Lisp syntax or evaluator behaviour
- No redesign of CodeMirror UI features
- No bundle-size optimization beyond avoiding unnecessary duplication

---

## Open questions

1. **Namespace path:** should the converted files reuse the existing
   `app/src/repulse/lisp-lang/` directory or move to a CLJS-friendly namespace
   such as `repulse.lisp-lang-support` / `repulse.lisp_lang`? Recommendation:
   keep generated Lezer JS under `lisp-lang/`, and put converted CLJS support in
   a namespace that Shadow handles cleanly.
2. **Data reuse:** should hover docs and completion docs become one normalized
   CLJS data source? Recommendation: yes, if it reduces duplication without
   expanding the phase into a documentation rewrite.
